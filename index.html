    Engagement

    This module handles the creation, updating and deletion of engagements. The following represents the different options for an engagement:

    Culture

    Culture Value
    christian 0
    hindu 1
    muslim 2
    traditional 3

    Privacy

    Privacy Value
    private 0
    invited 1
    public 2

    For privacy, the invited value indicates that only the invited guests have access to seeing this engagement Note: for an engagement to be created, we only require to have a couple and these two have to have created an account with us.

    Get one public engagement by id.

    Returns a public engagement given its id as a parameter.

    Example request:

    curl -X POST "http://localhost:8000//api/engagement/get_one" \
-H "Accept: application/json" \
    -d "engagement_id"="772121753" \

    var settings = {
    "async": true,
    "crossDomain": true,
    "url": "http://localhost:8000//api/engagement/get_one",
    "method": "POST",
    "data": {
        "engagement_id": 772121753
},
    "headers": {
        "accept": "application/json"
    }

    2. Authentication - with Facebook ID or email and password.
    3. User details - access to logged in user's details.

    Please ensure you have the space after the 'Bearer ' value of the 'Authorization' header e.g. 'Bearer eyJ0eXAiOiJKV1QiL...'
